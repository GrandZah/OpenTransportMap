# Описание data_transport

Получаем данные из OSM, парсим, преобразуем в стандартизованный вид.


Работаем с данными в формате EPSG:4326 - как получаем от OSM.

- build_edges.py - преобразуем табличные данные с маршрутами и дистанциями между остановками в таблицу, которая хранит представление о транспортной схеме в виде графа. Каждая строка содержит информацию про ребро связи между двумя остановками для конкретного маршрута, а также порядок обхода. 

- **get_bus_layers.py** - основной pipeline папки. Получаем данные - парсим - преобразуем - складываем. Также кэшируем данные - чтобы лишний раз не скачивать с интернета.
- main.py - точка входа для того, чтобы посмотреть как работает только данная папка (без визуализации). Для отладки процесса получения данных
- **parser_osm.py** - основной код - парсим данные с OSM - обрабатываем ошибки (их может быть много) и возвращаем данные о маршрутах, остановках.
- city_route_database.py - формализуем в каком формате возвращаются данные после парсинга и преобразования. Также переделываем в удобные и быстрые структуры данных - для поисков в момент отображения
